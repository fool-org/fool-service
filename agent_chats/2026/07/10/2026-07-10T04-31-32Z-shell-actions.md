# Vue Shell Actions

## Prompt
- Continue the Docker/FoolFrame/Vue migration toward a usable frontend while
  keeping View metadata ahead of data, reusing shared helpers, and controlling
  file size and duplication.

## Scope
- Compared the old `default.jade`, `tbar.jade`, `message.js`, and
  `showerror.js` shell behavior with the Vue app.
- Added a focused `ShellActions.vue` topbar component for the signed-in user,
  message count/popover, refresh, and logout.
- Moved `getuserinfo`, `getmsg`, and `getnotify` loading out of API Tools and
  into shared shell initialization plus a guarded 15-second poll.
- Kept the last non-empty message result across empty polls so users can open
  it, while reusing the existing View-first list/detail loaders for message
  targets.
- Rendered notification counts beside matching metadata-driven shell menus.
- Added shared Pascal/camel user, message-time, and notification helpers.
- Made the message popover fit desktop and 390px viewports.
- Reseeded the runtime-doctor message immediately before its destructive
  `getmsg` assertion to avoid races with the live Vue poll.

## Risks
- The full legacy login flow still lives in API Tools rather than a dedicated
  signed-out Vue screen; this shell slice only makes the authenticated state
  user-facing.
- Messages remain in the current browser session until logout or reload; the
  backend still owns the legacy destructive-poll state.
