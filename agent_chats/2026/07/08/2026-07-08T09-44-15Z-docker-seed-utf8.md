# Docker seed UTF-8 metadata

## Prompt

Continue the FoolFrame migration and keep the Vue front end usable without
binding page rendering to concrete business DTOs.

## Scope

- Added explicit `SET NAMES utf8mb4` to Docker seed SQL files that contain
  legacy Chinese metadata.
- Replayed `006-view.sql` and `010-query.sql` into the current Docker MySQL
  volume with `--default-character-set=utf8mb4` so existing local data was
  repaired.
- Tightened `scripts/runtime_doctor.py` so the default View smoke fails if
  operation labels regress to mojibake.
- Updated `tasks.md` and `docs/migration/foolframe-parity.md`.

## Validation

- `python scripts/runtime_doctor_test.py`: passed, 13 tests.
- `docker compose exec -T mysql mysql --default-character-set=utf8mb4 ...`:
  `SW_SYS_VIEW_OPERATION` now stores `删除` as `E588A0E999A4` and `保存` as
  `E4BF9DE5AD98`; `SE_COMPARETYPE` labels also read as UTF-8.
- `python scripts/runtime_doctor.py`: passed, including the new operation
  label check.
- Browser check at `http://localhost:8081/`: visible row-operation buttons
  show `删除` / `保存`; mojibake strings are absent.

## Risks

- Existing external Docker volumes created before this fix need the same seed
  replay or a fresh volume to repair already-corrupted text rows.
