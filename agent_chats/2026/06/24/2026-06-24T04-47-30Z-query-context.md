# Query Context Parity

## Scope

- Compared legacy `../FoolFrame/src/Server/SWDQ01-Soway.Query/QueryContext.cs`.
- Migrated the orchestration layer that owns the active query instance, adds the first or joined table, resets state, and delegates paged execution.
- Kept report/saved-query surfaces out of this slice; this change is limited to `QueryContext` behavior around `QueryInstance`, `SelectedTables`, and `JdbcQueryExecutor`.

## Changes

- Added `QueryContext` with `add(QueryTable)`, `add(QueryTable, SelectedTable)`, `clear()`, `getResult(int)`, and `getResult(int, int)`.
- Preserved legacy selected-table aliases by using `QueryTable.showName` for the first table and appending the current duplicate count for joined tables.
- `clear()` replaces the current `QueryInstance` instead of mutating the old instance in place.
- `getResult()` delegates to the configured `JdbcQueryExecutor` using the current instance.
- Updated `docs/migration/foolframe-parity.md` to record `QueryContext` orchestration and bump `fool-query` to 40 Java main files.
